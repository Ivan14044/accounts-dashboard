# Spec Delta: Account Management

## MODIFIED Requirements

### Requirement: Статистика аккаунтов
Система SHALL предоставлять статистическую информацию об аккаунтах с оптимальной производительностью.

**Изменение:** Оптимизация с 8 запросов к БД до 1 запроса с использованием SQL агрегации.

#### Scenario: Получение общей статистики
- **WHEN** пользователь открывает дашборд
- **THEN** система выполняет 1 SQL запрос с агрегацией
- **AND** возвращает общее количество аккаунтов
- **AND** возвращает распределение по статусам
- **AND** возвращает количество с email и 2FA
- **AND** возвращает количество изменений за 24 часа
- **AND** время выполнения <150ms

#### Scenario: Получение статистики с фильтром
- **WHEN** пользователь применяет фильтр к аккаунтам
- **THEN** система пересчитывает статистику для фильтрованных данных
- **AND** использует тот же оптимизированный запрос с WHERE условием
- **AND** время выполнения <200ms

#### Scenario: Кэширование статистики
- **WHEN** статистика запрашивается повторно в течение 5 минут
- **THEN** система возвращает закэшированный результат
- **AND** не выполняет запрос к БД
- **AND** время ответа <10ms

#### Scenario: Инвалидация кэша при изменениях
- **WHEN** пользователь изменяет данные аккаунта (статус, удаление)
- **THEN** система инвалидирует кэш статистики
- **AND** следующий запрос выполняет свежий расчёт

## ADDED Requirements

### Requirement: Константы конфигурации
Система SHALL использовать именованные константы вместо magic numbers.

#### Scenario: Использование константы размера страницы
- **WHEN** система выполняет пагинацию
- **THEN** использует `Config::DEFAULT_PAGE_SIZE` вместо числа 100
- **AND** позволяет легко изменить размер через Config

#### Scenario: Использование константы лимитов
- **WHEN** система проверяет лимиты запросов
- **THEN** использует `Config::MAX_IDS_PER_REQUEST` вместо числа 50000
- **AND** обеспечивает читабельность кода

#### Scenario: Использование константы таймаутов
- **WHEN** система работает с кэшем
- **THEN** использует `Config::CACHE_TTL` для времени жизни
- **AND** централизует конфигурацию
