# Change: Полная переработка таблицы аккаунтов

## Why
- Текущая верстка накопила противоречивые стили (10+ CSS файлов с !important), что приводит к постоянным визуальным багам и перекрытиям.
- Пользователь требует переписать таблицу с нуля, чтобы обеспечить стабильную работу sticky колонок, сортировки, виртуализации и массовых действий.
- Существующая структура HTML и JS сложно поддерживается: inline-фрагменты, дублирование логики, отсутствие четких компонентных границ.

## What Changes
- Спроектировать и реализовать полностью новый шаблон таблицы в 	emplates/dashboard.php с отдельными зонами: тулбар, шапка, тело, футер.
- Создать модульную CSS-архитектуру (ядро + light overrides) без конфликтов с unified-theme, используя CSS Custom Properties и BEM-подобные классы.
- Переписать JS-слой таблицы: менеджер разметки, сортировка, виртуализация, массовые действия, sticky-scrollbar — объединить в TableModule с четким API.
- Обеспечить полноценную адаптивность (desktop/tablet/mobile) и отказоустойчивость (fallback без JS, graceful degradation).
- Подготовить финальную сборку assets и инструкцию по миграции.

## Impact
- Affected specs: dashboard-table (новый capability).
- Affected code: 	emplates/dashboard.php, ssets/css/table-*.css, ssets/js/table-*.js, ssets/build/*, sticky scrollbar и автообновление.
- **BREAKING:** верстка таблицы и публичный JS API меняются — все внешние скрипты нужно адаптировать.
