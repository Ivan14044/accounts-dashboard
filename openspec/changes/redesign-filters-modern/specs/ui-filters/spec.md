# Spec: UI Filters

## ADDED Requirements

### Requirement: Визуальные chips для активных фильтров
Система SHALL отображать активные фильтры в виде визуальных chips (таблеток) для быстрой идентификации и удаления.

#### Scenario: Отображение активного фильтра как chip
- **WHEN** пользователь активирует фильтр (например, "Email")
- **THEN** под заголовком появляется chip "✓ Email"
- **AND** chip имеет кнопку удаления (крестик)
- **AND** chip окрашен в primary цвет

#### Scenario: Удаление фильтра через chip
- **WHEN** пользователь кликает на крестик в chip
- **THEN** фильтр деактивируется
- **AND** chip исчезает с анимацией
- **AND** таблица обновляется

#### Scenario: Множественные активные фильтры
- **WHEN** активно несколько фильтров
- **THEN** все отображаются как chips в одной строке
- **AND** chips переносятся на новую строку при нехватке места
- **AND** максимум 10 chips видимо, остальные в "+ N ещё"

### Requirement: Современное поле поиска
Система SHALL предоставлять улучшенное поле поиска с визуальными индикаторами.

#### Scenario: Поле поиска с иконками
- **WHEN** пользователь видит поле поиска
- **THEN** слева отображается иконка лупы
- **AND** справа кнопка очистки (когда есть текст)
- **AND** поле имеет увеличенный размер и padding

#### Scenario: Очистка поиска
- **WHEN** пользователь вводит текст в поиск
- **THEN** справа появляется кнопка очистки
- **WHEN** пользователь кликает на кнопку очистки
- **THEN** текст удаляется
- **AND** фильтр сбрасывается

#### Scenario: Focus эффект
- **WHEN** пользователь фокусируется на поле поиска
- **THEN** появляется shadow вокруг поля
- **AND** border меняет цвет на primary
- **AND** transition плавный (300ms)

### Requirement: Кнопки статусов вместо dropdown
Система SHALL отображать популярные статусы как визуальные кнопки для быстрого доступа.

#### Scenario: Отображение топ-5 статусов
- **WHEN** пользователь видит блок фильтров
- **THEN** топ-5 статусов отображаются как кнопки
- **AND** каждая кнопка показывает название и количество записей
- **AND** остальные статусы скрыты в "+ Ещё N"

#### Scenario: Выбор статуса кнопкой
- **WHEN** пользователь кликает на кнопку статуса
- **THEN** кнопка становится active (другой стиль)
- **AND** фильтр применяется
- **AND** таблица обновляется
- **AND** создаётся chip в секции активных фильтров

#### Scenario: Множественный выбор статусов
- **WHEN** пользователь выбирает несколько статусов
- **THEN** все выбранные кнопки имеют active состояние
- **AND** создаётся один chip "Статусы: N выбрано"

### Requirement: Toggle switches для быстрых фильтров
Система SHALL использовать современные toggle switches вместо чекбоксов для быстрых фильтров.

#### Scenario: Отображение toggle switch
- **WHEN** пользователь видит быстрые фильтры (Email, 2FA, Token)
- **THEN** каждый отображается как iOS-style toggle switch
- **AND** switch имеет иконку и label
- **AND** активное состояние окрашено в success цвет

#### Scenario: Переключение toggle
- **WHEN** пользователь кликает на toggle
- **THEN** switch анимированно переключается
- **AND** фильтр применяется/снимается
- **AND** chip добавляется/удаляется
- **AND** transition занимает 200ms

### Requirement: Группировка фильтров по категориям
Система SHALL группировать фильтры по логическим категориям для лучшей организации.

#### Scenario: Основные фильтры всегда видимы
- **WHEN** блок фильтров открыт
- **THEN** всегда видны: Поиск, Статусы, Быстрые фильтры
- **AND** секция занимает минимум места

#### Scenario: Дополнительные фильтры в collapse
- **WHEN** пользователь кликает "▼ Дополнительные фильтры"
- **THEN** раскрывается панель с диапазонами
- **AND** показываются поля: Pharma, Друзья, Год, Limit RK
- **AND** анимация плавная (slide-down)

### Requirement: Анимации и transitions
Система SHALL использовать плавные анимации для всех взаимодействий.

#### Scenario: Анимация появления chips
- **WHEN** активируется фильтр
- **THEN** chip появляется с fade-in анимацией
- **AND** слегка увеличивается (scale 0.8 → 1.0)
- **AND** duration 300ms

#### Scenario: Анимация удаления chips
- **WHEN** пользователь удаляет chip
- **THEN** chip исчезает с fade-out
- **AND** уменьшается (scale 1.0 → 0.8)
- **AND** соседние chips сдвигаются плавно

#### Scenario: Ripple effect на кнопках
- **WHEN** пользователь кликает на кнопку статуса
- **THEN** появляется ripple эффект от точки клика
- **AND** эффект распространяется концентрически
- **AND** затухает через 400ms

### Requirement: Адаптивность для мобильных
Система SHALL оптимизировать фильтры для мобильных устройств.

#### Scenario: Вертикальный layout на мобильных
- **WHEN** ширина экрана < 768px
- **THEN** все фильтры располагаются вертикально
- **AND** поле поиска занимает всю ширину
- **AND** кнопки статусов стакаются вертикально

#### Scenario: Touch-friendly элементы
- **WHEN** устройство с touch-экраном
- **THEN** все интерактивные элементы имеют минимум 44px высоту
- **AND** paddings увеличены для лёгкого нажатия
- **AND** ripple эффекты заметнее

## Performance Targets

- Анимации: 60 FPS
- Отклик на клик: < 100ms
- Загрузка стилей: < 50ms
- Общий вес CSS: < 20KB

## Compatibility

- Chrome/Edge: 90+
- Firefox: 88+
- Safari: 14+
- Mobile browsers: iOS Safari 14+, Chrome Mobile 90+
