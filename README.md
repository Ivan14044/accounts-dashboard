# 🚀 Accounts Dashboard - Система управления аккаунтами

Современная веб-система для управления аккаунтами с расширенными возможностями фильтрации, поиска, массовых операций и аналитики.

## ✨ Основные возможности

- **Управление аккаунтами** - полный CRUD функционал с inline-редактированием
- **Продвинутая фильтрация** - множественные фильтры с сохранением пресетов
- **Быстрый поиск** - поиск по всем полям с подсветкой результатов
- **Массовые операции** - обновление статусов, удаление, перенос аккаунтов
- **Сортировка** - по любому полю с поддержкой числовых типов
- **Экспорт данных** - CSV/JSON экспорт с применением текущих фильтров
- **История изменений** - полный аудит всех операций с аккаунтами
- **Корзина** - soft delete с возможностью восстановления
- **Избранное** - быстрый доступ к важным аккаунтам
- **Сохраненные фильтры** - пресеты для часто используемых запросов
- **Виртуализация таблицы** - оптимизация для работы с большими объемами данных
- **Кастомные карточки** - настраиваемая статистика на главной странице

## 📋 Требования

- **PHP** 7.4 или выше
- **MySQL** 5.5+ или MariaDB 10.0+
- **Apache** с mod_rewrite или **Nginx**
- **Расширения PHP:** mysqli, json, mbstring

## 🚀 Быстрый старт

### 1. Установка

```bash
# Клонируйте репозиторий или загрузите файлы
git clone <repository-url>
cd dashboard

# Создайте файл конфигурации
cp .env.example .env
# Отредактируйте .env с вашими данными БД
```

### 2. Настройка базы данных

Создайте файл `.env` в корне проекта:

```php
DB_HOST=localhost
DB_NAME=your_database
DB_USER=your_username
DB_PASS=your_password
DB_PORT=3306
```

Или используйте `config.local.php`:

```php
<?php
$DB_HOST = 'localhost';
$DB_NAME = 'your_database';
$DB_USER = 'your_username';
$DB_PASS = 'your_password';
$DB_PORT = 3306;
```

### 3. Создание таблиц

Таблицы создаются автоматически при первом запуске. Если нужно создать вручную:

```bash
# Через MySQL CLI
mysql -u username -p database_name < sql/create_all_tables.sql

# Или через phpMyAdmin
# Загрузите и выполните sql/create_all_tables.sql
```

### 4. Применение индексов (рекомендуется)

Для оптимальной производительности примените индексы:

```bash
# Автоматически (безопасный способ)
php apply_indexes_safe.php

# Или вручную через phpMyAdmin
# Загрузите sql/performance_indexes.sql
```

### 5. Настройка пользователей

Создайте файл `users.json` (пример):

```json
{
  "admin": {
    "password": "$2y$10$...",
    "role": "admin"
  }
}
```

Используйте `php -r "echo password_hash('your_password', PASSWORD_BCRYPT);"` для генерации хеша пароля.

### 6. Запуск

Откройте в браузере:

```
http://localhost/dashboard/
```

## 🎯 Основные функции

### Inline-редактирование

Редактируйте поля прямо в таблице:
- Кликните на поле для редактирования
- Введите новое значение
- Нажмите Enter или кликните вне поля для сохранения
- Автоматическая валидация типов данных

### Фильтрация

- **Текстовый поиск** - поиск по всем полям
- **Фильтр по статусам** - множественный выбор
- **Фильтр по валюте** - выбор валют
- **Фильтр по геолокации** - выбор стран/регионов
- **Диапазоны значений** - для числовых полей
- **Булевы фильтры** - есть/нет email, 2FA, токен и т.д.
- **Сохранение фильтров** - создание пресетов для быстрого доступа

### Массовые операции

- Обновление статусов для выбранных аккаунтов
- Массовое удаление (в корзину)
- Массовый перенос аккаунтов
- Массовое обновление полей

### Экспорт

- Экспорт в CSV с применением текущих фильтров
- Экспорт в JSON
- Настраиваемые колонки для экспорта

## 🔧 Настройка

### Конфигурация производительности

В `includes/Config.php` можно настроить:

```php
// Кэширование
const CACHE_TTL_STATS = 300; // 5 минут
const CACHE_TTL_METADATA = 3600; // 1 час

// Rate limiting
const RATE_LIMIT_LOGIN = 5; // запросов в минуту
const RATE_LIMIT_API = 100; // запросов в минуту

// Сессии
const SESSION_TIMEOUT = 3600; // 1 час
const SESSION_ACTIVITY_UPDATE_INTERVAL = 300; // 5 минут
```

### Сборка ресурсов

Для продакшена соберите CSS/JS:

```bash
php build_assets.php
```

Результат будет в `assets/build/dashboard.min.css` и `dashboard.min.js`.

## 🛡️ Безопасность

- **CSRF защита** - все POST/PUT/DELETE запросы защищены токенами
- **Rate limiting** - защита от злоупотреблений API
- **Prepared statements** - защита от SQL injection
- **XSS защита** - экранирование всех пользовательских данных
- **Валидация входных данных** - проверка типов и форматов
- **Аудит логирование** - запись всех изменений
- **Маскирование чувствительных данных** - пароли и токены не логируются в открытом виде

## 📊 Производительность

- **Виртуализация таблицы** - рендеринг только видимых строк
- **Кэширование запросов** - кэш метаданных БД и статистики
- **Индексы БД** - оптимизированные запросы
- **Минификация ресурсов** - объединенные CSS/JS файлы
- **HTTP кэширование** - ETag и условные запросы
