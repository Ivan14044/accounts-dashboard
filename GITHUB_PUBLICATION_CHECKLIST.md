## 🚨 Критические действия перед коммитом

### ДЕЙСТВИЕ 1: Проверить локальные файлы

Убедитесь, что эти файлы НЕ будут закоммичены:

```bash
# Проверить статус
git status

# Если видите .env, config.local.php или users.json в "Changes to be committed":
git reset HEAD .env config.local.php users.json
```

---

### ДЕЙСТВИЕ 2: Проверить историю Git (если репозиторий уже существует)

```bash
# Проверить, не были ли закоммичены чувствительные данные ранее
git log --all --full-history -- .env
git log --all --full-history -- config.local.php
git log --all --full-history -- users.json

# Если файлы найдены в истории, нужно удалить их:
# ВНИМАНИЕ: Это перепишет историю Git!
git filter-branch --force --index-filter \
  "git rm --cached --ignore-unmatch .env config.local.php users.json" \
  --prune-empty --tag-name-filter cat -- --all
```

---

### ДЕЙСТВИЕ 3: Финальная проверка перед push

```bash
# 1. Проверить, что чувствительные файлы не в индексе
git ls-files | grep -E "\.env$|config\.local\.php$|users\.json$"

# 2. Проверить статус
git status

# 3. Если все ОК, можно коммитить
git add .
git commit -m "Initial commit"
git push origin main
```
