## 📑 Сохранённые фильтры (Presets)

### Что это?

Сохранённые фильтры позволяют сохранить набор параметров фильтрации для быстрого доступа в будущем. Вместо того чтобы каждый раз вручную настраивать все фильтры, вы можете сохранить их с именем и применить одним кликом.

### Где находится?

Выпадающий список **"Сохранённые фильтры"** находится в **заголовке блока фильтров** на главной странице дашборда (в правом верхнем углу панели фильтров).

---

### 🎯 Как использовать

#### Шаг 1: Применить фильтры

1. Откройте главную страницу дашборда
2. Нажмите на панель **"Фильтры"** (если она свёрнута)
3. Настройте нужные фильтры:
   - Поиск по тексту
   - Статусы
   - Валюта
   - Геолокация
   - Диапазоны значений
   - Булевы фильтры (есть/нет email, 2FA и т.д.)
4. Примените фильтры (они применятся автоматически при изменении)

#### Шаг 2: Сохранить фильтр

1. После применения фильтров в заголовке блока фильтров найдите выпадающий список **"Сохранённые фильтры"**
2. Нажмите на выпадающий список
3. Выберите пункт **"Сохранить текущий фильтр"**
4. В появившемся окне введите название для фильтра, например:
   - "Активные аккаунты"
   - "Аккаунты с email"
   - "Новые аккаунты 2025"
   - "Аккаунты для проверки"
5. Нажмите **"OK"** или **Enter**
6. Фильтр будет сохранён и появится в списке сохранённых фильтров

#### Шаг 3: Применить сохранённый фильтр

1. Откройте выпадающий список **"Сохранённые фильтры"**
2. Найдите нужный фильтр в списке
3. Нажмите на кнопку **"✓"** (галочка) рядом с названием фильтра
4. Фильтры применятся автоматически, и вы перейдёте на страницу с результатами

#### Шаг 4: Удалить сохранённый фильтр

1. Откройте выпадающий список **"Сохранённые фильтры"**
2. Найдите фильтр, который хотите удалить
3. Нажмите на кнопку **"🗑️"** (корзина) рядом с названием фильтра
4. Подтвердите удаление в появившемся окне
5. Фильтр будет удалён из списка

---

### 💡 Примеры использования

#### Пример 1: Сохранить фильтр для активных аккаунтов

1. Примените фильтры:
   - Статус: выберите "active" или другие активные статусы
   - Или поиск: "active"
2. Сохраните как **"Активные аккаунты"**
3. Теперь одним кликом можно быстро открыть все активные аккаунты

#### Пример 2: Сохранить фильтр для проверки аккаунтов

1. Примените фильтры:
   - Статус: выберите "new" или "pending"
   - Есть email: Да
   - Есть пароль: Да
2. Сохраните как **"Аккаунты для проверки"**
3. Используйте для быстрого доступа к аккаунтам, требующим проверки

#### Пример 3: Сохранить фильтр для конкретного региона

1. Примените фильтры:
   - Геолокация: выберите нужную страну/регион
   - Валюта: соответствующая валюта
2. Сохраните как **"Аккаунты [название региона]"**
3. Используйте для работы с аккаунтами определённого региона

---

### ⚙️ Технические детали

- **Хранение:** Фильтры сохраняются в таблице `saved_filters` в базе данных
- **Привязка:** Каждый пользователь видит только свои сохранённые фильтры
- **Формат:** Параметры фильтров сохраняются в формате JSON
- **Ограничения:** Нет ограничений на количество сохранённых фильтров

## 🗑️ Корзина (Soft Delete)

### Что это?

Корзина (Soft Delete) — это функция "мягкого удаления" аккаунтов. Вместо физического удаления записей из базы данных, аккаунты помечаются как удалённые и автоматически исключаются из всех выборок.

### Преимущества

- ✅ **Безопасность:** Данные не теряются при случайном удалении
- ✅ **Восстановление:** Возможность восстановить удалённые аккаунты
- ✅ **Аудит:** История удалений сохраняется
- ✅ **Производительность:** Удалённые записи автоматически исключаются из запросов

---

### 🎯 Как использовать

#### Удаление аккаунтов (в корзину)

1. На главной странице дашборда выберите аккаунты для удаления:
   - Отметьте чекбоксы рядом с нужными аккаунтами
   - Или выберите все отфильтрованные аккаунты (чекбокс в заголовке таблицы)
2. В панели инструментов нажмите кнопку **"Удалить"** (иконка корзины 🗑️)
3. Подтвердите удаление в появившемся окне
4. Аккаунты будут перемещены в корзину:
   - Они исчезнут из списка
   - Поле `deleted_at` в БД будет заполнено текущей датой и временем
   - Данные останутся в базе данных

#### Что происходит при удалении?

- ✅ Аккаунт **исчезает** из всех списков и выборок
- ✅ Аккаунт **исключается** из всех фильтров
- ✅ Аккаунт **не удаляется** физически из базы данных
- ✅ Дата и время удаления **сохраняется** в поле `deleted_at`
- ✅ История изменений **логируется** в audit log

#### Проверка удалённых аккаунтов в БД

Если нужно проверить удалённые аккаунты напрямую в базе данных:

```sql
-- Посмотреть все удалённые аккаунты
SELECT * FROM accounts WHERE deleted_at IS NOT NULL;

-- Посмотреть последние удалённые аккаунты
SELECT id, login, email, deleted_at 
FROM accounts 
WHERE deleted_at IS NOT NULL 
ORDER BY deleted_at DESC 
LIMIT 10;
```

---

### ⚠️ Важные замечания

1. **Автоматическое исключение:**
   - Удалённые аккаунты автоматически исключаются из всех выборок
   - Они не отображаются в таблице, статистике, фильтрах
   - Это работает "из коробки" - никаких дополнительных действий не требуется

2. **Восстановление (опционально):**
   - В текущей версии функция восстановления реализована на уровне API
   - Метод `AccountsService::restoreAccounts()` доступен для программистов
   - UI для восстановления можно добавить в будущем

3. **Физическое удаление:**
   - Если нужно окончательно удалить аккаунт из БД, это можно сделать вручную через SQL:
   ```sql
   DELETE FROM accounts WHERE id = ? AND deleted_at IS NOT NULL;
   ```
   - Или дождаться автоматической очистки корзины (если будет реализована)

---

### 🔧 Технические детали

#### Структура базы данных

```sql
-- Поле deleted_at добавлено в таблицу accounts
ALTER TABLE accounts 
ADD COLUMN deleted_at TIMESTAMP NULL DEFAULT NULL;

-- Индексы для оптимизации
CREATE INDEX idx_deleted_at ON accounts(deleted_at);
CREATE INDEX idx_status_deleted ON accounts(status, deleted_at);
```

#### Как работает автоматическое исключение

1. Все запросы к таблице `accounts` автоматически добавляют условие:
   ```sql
   WHERE (deleted_at IS NULL OR deleted_at = "")
   ```

2. Это происходит на уровне `FilterBuilder` и `AccountsService`

3. Удалённые записи не попадают в:
   - Список аккаунтов (`getAccounts()`)
   - Подсчёт записей (`getAccountsCount()`)
   - Статистику (`getStatistics()`)
   - Все фильтры и выборки

---

### 💡 Примеры использования

#### Пример 1: Удаление одного аккаунта

1. Откройте страницу просмотра аккаунта (`view.php?id=123`)
2. Вернитесь на главную страницу
3. Найдите аккаунт в таблице и отметьте чекбокс
4. Нажмите кнопку **"Удалить"**
5. Подтвердите удаление
6. Аккаунт исчезнет из списка, но данные останутся в БД

#### Пример 2: Массовое удаление по фильтру

1. Примените фильтры для выбора аккаунтов (например, статус "old")
2. Отметьте чекбокс **"Выбрать все отфильтрованные"**
3. Нажмите кнопку **"Удалить"**
4. Все выбранные аккаунты будут перемещены в корзину

#### Пример 3: Проверка удалённых аккаунтов

1. Подключитесь к базе данных
2. Выполните SQL запрос:
   ```sql
   SELECT COUNT(*) as deleted_count 
   FROM accounts 
   WHERE deleted_at IS NOT NULL;
   ```
3. Посмотрите список удалённых аккаунтов:
   ```sql
   SELECT id, login, email, status, deleted_at 
   FROM accounts 
   WHERE deleted_at IS NOT NULL 
   ORDER BY deleted_at DESC;
   ```

### 🔄 Восстановление из корзины (для разработчиков)

Если нужно восстановить аккаунт программно:

```php
// В PHP коде
$service = new AccountsService();
$restored = $service->restoreAccounts([123, 456]); // ID аккаунтов для восстановления
echo "Восстановлено аккаунтов: " . $restored;
```

Или напрямую в SQL:

```sql
-- Восстановить один аккаунт
UPDATE accounts 
SET deleted_at = NULL 
WHERE id = 123;

-- Восстановить несколько аккаунтов
UPDATE accounts 
SET deleted_at = NULL 
WHERE id IN (123, 456, 789);
```

## ❓ Часто задаваемые вопросы

### Сохранённые фильтры

**Q: Где хранятся сохранённые фильтры?**  
A: В таблице `saved_filters` в базе данных. Каждый пользователь видит только свои фильтры.

**Q: Можно ли поделиться фильтром с другим пользователем?**  
A: В текущей версии нет, но можно скопировать параметры из URL или создать такой же фильтр вручную.

**Q: Есть ли ограничение на количество сохранённых фильтров?**  
A: Нет, вы можете сохранять любое количество фильтров.

### Корзина

**Q: Удалённые аккаунты занимают место в базе данных?**  
A: Да, они занимают место, но исключены из всех выборок и не влияют на производительность.

**Q: Можно ли восстановить удалённые аккаунты через интерфейс?**  
A: В текущей версии нет UI для восстановления, но это можно сделать через SQL или добавить в будущем.

**Q: Как окончательно удалить аккаунт из базы данных?**  
A: Можно выполнить SQL запрос: `DELETE FROM accounts WHERE id = ? AND deleted_at IS NOT NULL`

**Q: Автоматически ли очищается корзина?**  
A: В текущей версии нет автоматической очистки, но можно добавить эту функцию в будущем (например, удаление записей старше 30 дней).
