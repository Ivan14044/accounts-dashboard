# Анализ проблемы с выбором строк в таблице

## Описание проблемы

Выбор строк на странице в таблице работает некорректно. Выбранные строки теряются при:
- Переходе на другую страницу (пагинация)
- Обновлении таблицы через AJAX
- Виртуализации таблицы (когда строки удаляются и добавляются обратно)

## Технический анализ

### Текущая архитектура

1. **Хранение выбранных ID**: 
   - Глобальная переменная `selectedIds` (Set)
   - Сохранение в `localStorage` через `saveSelectedIds()`
   - Загрузка из `localStorage` через `loadSelectedIds()`

2. **Восстановление состояния чекбоксов**:
   - Функция `initCheckboxStates()` - восстанавливает состояние чекбоксов из `selectedIds`
   - Вызывается в `afterRender()` после обновления таблицы
   - Вызывается в `updateVisibleRows()` при виртуализации

3. **Обработка событий**:
   - Событие `change` на чекбоксах → `toggleRowSelection()` → обновление `selectedIds`
   - Клик по строке → переключение чекбокса → обновление `selectedIds`

### Найденные проблемы

#### Проблема 1: Очистка `selectedIds` при пагинации

**Местоположение**: В коде есть места, где при изменении фильтров/пагинации очищается `selectedIds`:

```javascript
// При изменении фильтров - это ПРАВИЛЬНО
selectedAllFiltered = false; 
selectedIds.clear(); 
```

**Проблема**: При переходе на другую страницу (пагинация) НЕ должно очищаться `selectedIds`, так как выбранные строки могут быть на разных страницах.

**Решение**: Очищать `selectedIds` только при:
- Изменении фильтров (когда меняется набор видимых строк)
- Явном сбросе выбора пользователем
- НЕ очищать при простой пагинации

#### Проблема 2: Неправильная синхронизация при обновлении таблицы

**Местоположение**: `refreshDashboardData()` → `tableModule.updateRows()` → `afterRender()` → `initCheckboxStates()`

**Проблема**: Если `selectedIds` был очищен до обновления таблицы, то состояние не восстанавливается.

**Решение**: 
- Убедиться, что `loadSelectedIds()` вызывается при инициализации
- Убедиться, что `initCheckboxStates()` правильно восстанавливает состояние из `selectedIds`
- НЕ очищать `selectedIds` при пагинации

#### Проблема 3: Проблема с виртуализацией

**Местоположение**: `TableVirtualization.updateVisibleRows()` → `initCheckboxStates()`

**Проблема**: При виртуализации строки удаляются и добавляются обратно. Если `selectedIds` не синхронизирован, состояние теряется.

**Решение**: 
- `initCheckboxStates()` уже вызывается в `updateVisibleRows()` - это правильно
- Нужно убедиться, что `selectedIds` не очищается при виртуализации

#### Проблема 4: Неправильная логика в `initCheckboxStates()`

**Местоположение**: `templates/dashboard.php:4414-4431`

**Текущий код**:
```javascript
function initCheckboxStates() {
  document.querySelectorAll('.row-checkbox').forEach(cb => {
    const rowId = parseInt(cb.value);
    const isChecked = selectedAllFiltered || selectedIds.has(rowId);
    cb.checked = isChecked;
    const row = cb.closest('tr[data-id]');
    if (row) {
      updateRowSelectedClass(row, isChecked);
    }
  });
  // Обновление чекбокса "Выбрать все"
  const selectAllCheckbox = document.getElementById('selectAll');
  if (selectAllCheckbox) {
    const allCheckboxes = document.querySelectorAll('.row-checkbox');
    const checkedCheckboxes = document.querySelectorAll('.row-checkbox:checked');
    selectAllCheckbox.checked = allCheckboxes.length > 0 && allCheckboxes.length === checkedCheckboxes.length;
  }
}
```

**Проблема**: Логика выглядит правильной, но может быть проблема с порядком вызовов или с тем, что `selectedIds` очищается до вызова этой функции.
